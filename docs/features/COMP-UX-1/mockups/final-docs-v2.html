CORE-ASYNC-1 Design

Date: 2026-02-18
Feature ID: CORE-ASYNC-1
Canonical Plan: docs/features/CORE-ASYNC-1/plan.md

Problem

Current Python core/service runtime is largely synchronous, limiting concurrency and introducing event-loop blocking risk in async service contexts.

Goals

  1. Migrate core library from sync to async-native
  2. AsyncFalkorDBBackend validated for graph operations
  3. Maintain backward compatibility via sync wrappers
  4. Enable high-throughput pipeline processing

Non-Goals

  • Changing the public API surface
  • Breaking existing sync consumers

Architecture Options

Option A: Full async rewrite

  • Replace all I/O with async equivalents
  • Estimated 12-20 weeks

Option B: Incremental migration

  • Add async interfaces alongside sync
  • Deprecate sync over 2 releases
